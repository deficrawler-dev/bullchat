# BullChat

## Project

BullChat is a premium Web3 community platform built to become the home of X's crypto communities.

BullChat launches with a focused mission: creating the best community experience for Ansem's followers. It gives members a dedicated place to connect, collaborate, discover opportunities, and grow together beyond X.

As the platform evolves, BullChat will expand to support the broader crypto community on X and eventually multiple Web3 communities.

...

BullChat combines:

- Community chat
- Professional profiles
- Web3 networking
- Job board
- Token communities
- Direct messaging
- Community reputation
- Portfolio links
- Live market hubs

BullChat is NOT a meme coin website.

BullChat is NOT another Discord clone.

BullChat is a community operating system built for Web3.

# Mission

To become the home of Crypto Twitter communities by helping people connect, collaborate, discover opportunities, and build meaningful careers in Web3.

# Launch Strategy

## Phase 1

BullChat is built specifically for the Ansem community.

Every feature should improve the experience for Ansem's followers and make BullChat the place they naturally choose to hang out.

## Phase 2

Expand to the wider Crypto Twitter (CT) community while maintaining the same community-first philosophy.

## Phase 3

Allow other creators, founders, and Web3 communities to establish their own spaces on BullChat.

# Design Principles

Every screen should feel premium.

Take inspiration from:

- Linear
- Discord
- GitHub
- Notion
- X

Avoid:

- AI-generated layouts
- Colorful gradients
- Glassmorphism
- Floating blobs
- Illustrations
- Emojis
- Generic SaaS landing pages
- Unnecessary animations
- Visual clutter

Focus on:

- Excellent typography
- Clean spacing
- Clear hierarchy
- Accessibility
- Simplicity
- Consistency
- Fast interactions

Dark mode is the default experience.

Every design decision should prioritize usability over decoration.

# Coding Rules

- Never build features that were not requested.
- Never redesign completed pages.
- Never assume missing requirements.
- Ask questions instead of guessing.
- Write production-quality code.
- Keep components modular and reusable.
- Keep folders organized.
- Keep files maintainable.
- Never generate placeholder content unless requested.
- Explain important architectural decisions.
- Optimize for performance and scalability.

---

# User Experience

BullChat should feel:

- Fast
- Professional
- Premium
- Minimal
- Community-first

Every interaction should require as few clicks as possible.

Mobile-first design is required.

The onboarding process should be quick and frictionless while encouraging users to complete their profiles over time.

---

# Core Philosophy

Every feature added to BullChat should answer one question:

**Does this help the community connect, collaborate, or build?**

If the answer is no, the feature should be reconsidered.
